## 一、赛题速览

### 1.1 你要做什么

| 任务 | 输入 | 输出 | 评估 |
|------|------|------|------|
| **分类（主）** | 乳腺超声图像 | benign / malignant | AUC, Accuracy, Sensitivity, Specificity, Precision, F1 |
| **分割（辅）** | 同上 | 病灶区域二值掩膜 (0/1) | 辅助训练，不直接评分 |

核心场景：医生用超声探头扫描乳房，AI 需要判断结节是良性还是恶性，同时标出病灶区域。

### 1.2 为什么这个任务难

1. **超声图像噪声大**：散斑噪声（speckle noise）是超声的固有特性，低对比度
2. **样本量小**：训练集仅 1875 张（1507 训练 + 368 验证），属于深度学习中的"小数据"范畴
3. **类别不均衡**：良性:恶性 ≈ 2:1
4. **训练/测试分布偏移**：测试集图像尺寸是训练集的 **1.9 倍**（见下文 EDA）
5. **患者级数据泄露风险**：同一患者有多张不同角度的图像，划分不当会导致评估虚高

### 2.3 🚨 最重要发现：训练/测试分布严重偏移

| 统计量 | 训练集宽度 | 训练集高度 | 测试集宽度 | 测试集高度 |
|--------|-----------|-----------|-----------|-----------|
| mean | **321 px** | 391 px | **608 px** | 495 px |
| std | 77 | 53 | ~200 | ~100 |
| min/max | 240–579 | 268–478 | 190–1048 | 324–719 |

> 测试集图像平均宽度是训练集的 **1.9 倍**。这意味着模型在训练时从未见过 600px+ 宽度的图像，但测试时大量面对。这是所有"泛化差距"（Val AUC vs Test AUC 差约 0.02）的**物理根源**。

我们尝试过：
- 384×384 固定 Resize → 过拟合（AUC 0.888）
- RandomResizedCrop 多尺度训练 → 训练/测试管道不一致（AUC 0.877）
- 多尺度 TTA（推理时 0.75×/1.0×/1.25×）→ 非训练尺度的推理不靠谱（AUC 反降）

**最终选择：256×256 固定 Resize**。这是经过 5 个版本对比后的最优分辨率。虽然测试集信息被压缩更多，但更大的分辨率在 1507 样本上导致严重过拟合。

### 2.4 病灶形态分析

| 指标 | 良性 | 恶性 | 恶性/良性比 |
|------|------|------|-------------|
| BBOX 面积中位数 | 8,122 px² | 17,480 px² | 2.15× |
| 掩膜面积中位数 | 5,963 px² | 12,078 px² | 2.03× |
| 病灶占图比例均值 | 7.57% | 13.06% | 1.72× |

**恶性病灶平均大 2 倍**——但存在大量重叠，面积不能单独作为分类依据。模型需要学习病灶的**形态学特征**（边缘不规则、纵横比 >1、内部回声模式等）。

### 2.5 患者级划分：为什么必须做

训练集 1064 个 Case 中，**811 个有左右侧配对图像**（同一患者同时拍了左侧和右侧，或同一侧的不同角度）。如果按样本随机划分（V1 的做法），同一患者的左右侧图像可能分别进入训练集和验证集。模型学到的不是"病灶良恶性特征"，而是"这个患者的图像特征"——Val AUC 虚高到 0.959，但 Test AUC 只有 0.905。

**修复方法**（`get_case_split_train_val()`）：按 `Case` 字段分组 → shuffle Case 列表 → 前 20% Case 的全部图像进验证集，后 80% 进训练集。修复后 Val AUC 降到 0.924（诚实值），与 Test AUC 0.903 的差距缩小到 0.021。

**这是医学影像 AI 的黄金法则**：按患者/检查/序列划分，不能按单张图像划分。

### 2.6 BI-RADS 与恶性率的对应关系

| BI-RADS | 含义 | 训练集数量 | 恶性率 |
|---------|------|-----------|--------|
| 2 | 确定良性 | 111 | 1.8% |
| 3 | 可能良性 | 627 | 6.7% |
| 4 | 可疑恶性 | 860 | 59.6% |
| 5 | 高度怀疑恶性 | 277 | 97.5% |

**BI-RADS 4 是最大的灰色地带**（860 例，恶性率 60%），也是模型最需要发力的区域。V3.6 使用 BI-RADS 辅助预测头（让模型同时预测 BI-RADS 等级）获得了单模型最高的 F1（0.787）。

### 3.1 `src/data/dataset.py` — 数据加载与增强

这是项目的数据入口，修改次数最多（经过 V1/V2.3/V2.5/V6 多轮迭代）。

#### 核心类：`BreastUltrasoundDataset(Dataset)`

```python
class BreastUltrasoundDataset(Dataset):
    def __init__(self, data_dir, mode='train', transform=None, case_ids=None):
```

**参数说明：**
- `mode='train'`：从 CSV 读取，按 Case 过滤，返回 5 元组 `(image, mask, label, birads, filename)`
- `mode='test'`：遍历 `测试集/benign/` 和 `测试集/malignant/` 目录，自动匹配 `_mask.png`
- `case_ids`：患者级划分的核心参数。传入一组 Case ID，只加载这些 Case 的图像

**数据加载流程（`__getitem__`）：**

```
PIL.Image.open (RGB) + PIL.Image.open (L)
    ↓
np.array → Albumentations 增强 (图像+掩膜同步变换)
    ↓
PIL.fromarray → torchvision.transforms.ToTensor
    ↓
mask = (mask > 0.5).float()  # 确保二值 0/1
    ↓
返回: image[3,256,256], mask[1,256,256], label, birads_idx, filename
```

**增强管线（当前 V2.3 冠军版本）：**

```python
A.Compose([
    A.Resize(256, 256),                           # 固定尺寸, 与测试一致
    A.HorizontalFlip(p=0.5),                      # 左右翻转, 超声物理上等效
    A.Affine(translate_percent=±5%,               # 模拟探头微移
             scale=0.9~1.1, rotate=±15°, p=0.5),
    A.GaussNoise(std_range=10/255~50/255, p=0.3), # 模拟散斑噪声
    A.RandomBrightnessContrast(p=0.3),            # 模拟不同机器的增益
])
```

**增强设计原则：** 所有增强对良性和恶性样本一视同仁，不引入会导致误判的形变。水平翻转是唯一的方向增强（超声探头左右扫查等效），不做垂直翻转（探头方向固定）。

#### 关键函数：`get_case_split_train_val(data_dir, val_ratio=0.2, seed=42)`

```python
all_cases = df['Case'].unique().tolist()    # 1064 个 Case
random.shuffle(all_cases)                    # 随机打乱 Case
val_cases = set(all_cases[:n_val])          # 前 20% Case → 验证集
train_cases = set(all_cases[n_val:])        # 后 80% Case → 训练集
```

**为什么按 Case 划分而不是按样本划分：** 811/1064 个 Case 有左右侧配对图像。同一 Case 的多张图像在视觉上高度相关（同一患者的同一病灶从不同角度拍摄）。如果样本级随机划分，这些相关图像会分散到 train/val，模型靠患者特征而非病灶特征判断。修复前 Val AUC 虚高 0.035。

### 3.2 `src/models/base.py` — V2.3 冠军单模型

```python
class BreastCancerMultiTaskNet(nn.Module):
    def __init__(self, encoder_name="resnet34"):
        aux_params = dict(pooling='avg', dropout=0.4, activation=None, classes=1)
        self.model = smp.Unet(encoder_name=encoder_name,
                              encoder_weights="imagenet",
                              in_channels=3, classes=1,
                              aux_params=aux_params)

    def forward(self, x, birads=None):
        mask_logits, class_logits = self.model(x)
        return mask_logits, class_logits.squeeze(-1)
```

**架构图：**

```
输入 [B, 3, 256, 256]
    │
    ▼
ResNet34 Encoder (ImageNet 预训练)
    │
    ├──► U-Net Decoder ──► seg_head ──► mask_logits [B, 1, 256, 256]
    │                              ↕ BCEWithLogitsLoss (×1.5)
    │
    └──► GlobalAvgPool ──► Dropout(0.4) ──► Linear(512, 1) ──► cls [B]
                                                   ↕ FocalLoss(α=0.7, γ=2)
```

**关键设计决策：**

1. **为什么用 SMP 库而不是手写 U-Net？** SMP 提供 ImageNet 预训练的 ResNet34 encoder，在 1507 张超声图像上，预训练权重的迁移学习效果显著——从头训练 AUC 会低约 0.02-0.03。

2. **为什么用 `aux_params` 而不是独立的分类头？** SMP 的内置分类头（全局平均池化 + Dropout + Linear）经过了充分测试，与 U-Net decoder 配合默契。V3 系列尝试过独立分类头（手动从 encoder 最深特征提取），但 AUC 反而略低。

3. **为什么 `birads` 参数不参与前向计算？** 测试集没有 BI-RADS 标注（用占位符 0 替代）。如果让模型依赖 BI-RADS 输入做分类，测试集上所有 BI-RADS=0 的样本会被系统性误判。**BI-RADS 只能作为辅助预测目标（V3.6），不能作为输入特征（V2 的失败教训）。**

### 3.3 `src/models/v6.py` — V6 多尺度分割模型

这是为 LoMix 实验创建的多尺度输出模型。虽然实验最终失败了（AUC 0.872），但架构本身是有价值的参考。

```python
class BreastCancerMultiTaskNetV6(nn.Module):
    def __init__(self, encoder_name="resnet34"):
        # 5 个尺度分割头 (1×1 Conv → 1 通道)
        self.seg_heads = nn.ModuleList([
            nn.Conv2d(ch, 1, kernel_size=1) for ch in [256, 128, 64, 32, 16]
        ])
        # 独立的分类头
        self.cls_head = nn.Sequential(Dropout(0.4), Linear(512, 1))
```

**forward 的关键逻辑——手动遍历 decoder.blocks：**

```python
def forward(self, x, birads=None):
    features = self.seg_model.encoder(x)
    # features: [raw(3,256), stem(64,128), s1(64,64), s2(128,32), s3(256,16), s4(512,8)]
    
    # 获取空间尺寸用于上采样目标
    spatial_shapes = [f.shape[2:] for f in features][::-1]  # 反转: 最深→最浅
    
    # 移除 raw_input 和 stem, 反转
    enc_feats = features[1:][::-1]  # [s4(512,8), s3(256,16), s2(128,32), s1(64,64), stem(64,128)]
    
    head = enc_feats[0]  # 最深层 s4
    x_dec = self.seg_model.decoder.center(head)  # 瓶颈处理
    
    seg_logits_all = []
    for i, block in enumerate(self.seg_model.decoder.blocks):  # 5 个 block
        h, w = spatial_shapes[i + 1]  # 上采样目标尺寸
        skip = enc_feats[i + 1] if i < len(enc_feats) - 1 else None
        x_dec = block(x_dec, h, w, skip_connection=skip)
        
        logit = self.seg_heads[i](x_dec)  # 1×1 Conv → 1 通道
        logit = F.interpolate(logit, size=(256,256), mode='bilinear')
        seg_logits_all.append(logit)
    
    seg_logits_list = seg_logits_all[-4:]  # 取后 4 个尺度, 丢弃最粗糙的 8×8
    
    # 分类头
    deepest = features[-1]  # s4 特征 [B, 512, 8, 8]
    visual = self.pool(deepest).view(B, -1)
    cls_logits = self.cls_head(visual).squeeze(-1)
    
    return seg_logits_list, cls_logits
```

**为什么这样遍历 decoder.blocks？** SMP 的 DecoderBlock.forward 需要显式的 `(feature_map, target_height, target_width, skip_connection)` 参数。我们不能直接调用 `self.seg_model.decoder(*features)` 因为那样只返回最终输出。手动遍历让我们捕获每个中间 block 的输出。

**为什么取后 4 个尺度？** 5 个 decoder block 产生从 16×16 到 256×256 的 5 个中间输出。第一个 block（16×16）过于粗糙，丢弃后剩下 4 个尺度（32×32, 64×64, 128×128, 256×256），与 LoMix 论文配置一致。

### 3.4 `src/losses/lomix.py` — LoMix 多尺度融合损失模块

从 LoMix NeurIPS 2025 论文的 `trainer.py` 提取，适配为二分类分割。

#### BinaryDiceLoss

```python
class BinaryDiceLoss(nn.Module):
    def forward(self, logits, targets):
        probs = torch.sigmoid(logits)
        probs = probs.view(B, -1)      # 展平
        targets = targets.view(B, -1)
        intersect = (probs * targets).sum(dim=1)
        union = (probs²).sum(dim=1) + (targets²).sum(dim=1)
        dice = (2*intersect + smooth) / (union + smooth)
        return (1 - dice).mean()
```

**为什么修改原始的多类 DiceLoss？** 原始版本使用 one-hot 编码支持多类（如 8 个器官类别的 CT 分割）。我们的分割是单类二值（病灶/背景），只需要 sigmoid 后的单通道概率图。

#### WeightedFusion — 逐像素注意力融合

```python
class WeightedFusion(nn.Module):
    def __init__(self, num_stages, num_classes):
        self.weight_convs = nn.ModuleList([
            nn.Conv2d(num_classes, 1, kernel_size=1) for _ in range(num_stages)
        ])
    
    def forward(self, predictions):  # predictions: list of [B,C,H,W]
        weight_maps = [conv(p) for p in predictions]      # 每个尺度生成空间权重图
        weights = torch.stack(weight_maps, dim=1)          # [B, N, 1, H, W]
        weights = F.softmax(weights, dim=1)                # 逐像素归一化
        preds = torch.stack(predictions, dim=1)            # [B, N, C, H, W]
        fused = torch.sum(weights * preds, dim=1)          # 加权求和
        return fused
```

**这个模块做了什么？** 假设你有 3 个尺度的分割预测（32×32、64×64、128×128 上采样后的结果），WeightedFusion 为每个像素位置学习"应该更信任哪个尺度的预测"。在病灶边缘（需要细节），高分辨率尺度权重高；在病灶中心（需要全局判断），低分辨率尺度可能更可靠。

#### CombinatorialMutationsLossModule — 核心损失模块

```
输入: 4 个尺度的 seg_logits (全部上采样到 256×256) + ground truth mask

步骤 1: 深度监督 (Deep Supervision)
    对每个原始尺度 logit → BCE + Dice → softplus(α_i) × loss
    4 个尺度各有一个可学习权重 α_i

步骤 2: 组合突变 (Combinatorial Mutations)
    对 4 个尺度的所有 ≥2 的组合 (C(4,2)+C(4,3)+C(4,4)=11 种)
    每种组合用 4 种算子融合:
      - add: 求和（OR 逻辑, 置信度高）
      - mul: 相乘（AND 逻辑, 过滤分歧）
      - concat: 拼接后 1×1 Conv（学习最佳线性组合）
      - weighted_fusion: 逐像素注意力融合（空间自适应）
    每种融合结果 → BCE + Dice → softplus(β_j) × loss
    
    总计 4 (原始) + 11×4 (融合) = 48 个可学习权重, 15 个损失项

步骤 3: 软权重学习
    w = softplus(θ)  # θ 初始化为 0, softplus(0)≈0.693
    # 不用 softmax! 因为 softmax 强制权重和为 1, 造成竞争耦合
    # softplus 让每个权重独立 (0, ∞), 没用的会被推低→接近0 (自动剪枝)
    # 有用的可以无限增长

步骤 4: 返回 total_loss
    final_loss = deep_supervision_loss + mutation_loss
```

**为什么这个模块在 1507 样本上失败了？**
训练日志显示：所有权重从初始的 ~0.693（softplus(0)）被持续推低到 ~0.62。这是模型在"抗议"——它试图降低所有多尺度监督的贡献，但 15 个损失项的梯度总和仍然淹没了分类头。Val AUC 0.933（历史第三高）但 Test AUC 仅 0.872（历史倒数），泛化差距 0.061 是 V2.3 的 3 倍。

**这个模块什么时候有用？** 在 LoMix 原始论文的 Synapse 多器官 CT 数据集（数千张 3D 扫描）上，多尺度融合显著提升了小器官的分割。但我们的数据量和任务性质（2D 超声 + 分类为主）不匹配。保留此模块作为未来大数据量场景的参考。

#### 关键设计模式

**1. 双参数优化器（V6 LoMix 的核心）**

```python
# 损失模块有自己的可学习参数!
loss_module = CombinatorialMutationsLossModule(...)

# 优化器必须同时包含模型参数和损失模块参数
optimizer = optim.AdamW(
    list(model.parameters()) + list(loss_module.parameters()),
    lr=1e-4, weight_decay=1e-4
)
```

**为什么？** LoMix 的 48 个融合权重（α 和 β）需要在训练中自主学习。它们不是模型的参数（不参与前向推理），而是损失函数的参数（只在训练时起作用）。梯度通过 `loss.backward()` 同时更新模型权重和融合权重。

**2. AMP 自动混合精度**

```python
scaler = torch.amp.GradScaler('cuda')
with torch.amp.autocast('cuda'):
    seg_logits_list, cls_logits = model(images)
    loss = criterion_cls(cls_logits, labels) + 1.5 * loss_seg
scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
```

**为什么用 AMP？** RTX 4060 Laptop 只有 8GB VRAM。AMP 将前向传播中的矩阵运算从 FP32 降为 FP16，节省约 40% 显存，同时几乎不影响精度。注意：`torch.amp.autocast('cuda')` 是 PyTorch 2.4 的新 API（旧版是 `torch.cuda.amp.autocast()`）。

**3. FocalLoss 实现**

```python
class FocalLoss(nn.Module):
    def __init__(self, alpha=0.75, gamma=2.0):
        # alpha=0.7 用于实际训练
    
    def forward(self, logits, targets):
        bce = F.binary_cross_entropy_with_logits(logits, targets, reduction='none')
        pt = torch.exp(-bce)                         # 预测正确的概率
        alpha_t = targets*self.alpha + (1-targets)*(1-self.alpha)  # 类别权重
        return (alpha_t * (1-pt)**self.gamma * bce).mean()
```

**FocalLoss 的三个参数如何选择：**
- `α=0.7`：恶性样本权重 0.7，良性 0.3。反制类别不均衡（恶性仅占 32%）
- `γ=2`：调制难易样本。对于已经预测很好（pt→1）的样本，(1-pt)²→0，loss 极低；对于预测差（pt→0）的难样本，(1-pt)²→1，loss 接近完整 BCE
- 为什么不用 `pos_weight` BCE？FocalLoss 不仅加权类别，还区分难易样本——边界模糊的小病灶、等回声结节是"难样本"，标准 BCE 对它们和"易样本"一视同仁

**4. CosineAnnealingLR**

```python
scheduler = CosineAnnealingLR(optimizer, T_max=35, eta_min=1e-6)
```

学习率从 1e-4 余弦衰减到 1e-6。相比 StepLR（阶梯衰减），余弦衰减更平滑，在收敛末期的小学习率帮助模型进入更平坦的极小值。

### 3.6 `src/evaluation/test_single.py` — 单模型测试

```python
# TTA 水平翻转
cls_orig = model(images)           # 原图推理
cls_flip = model(flip(images))     # 翻转推理
final_prob = (sigmoid(cls_orig) + sigmoid(cls_flip)) / 2.0  # 平均

# 多阈值评估
for thresh in [0.40, 0.45, 0.50, 0.55]:
    preds = (probs > thresh).astype(int)
    auc = roc_auc_score(y_true, probs)        # AUC 与阈值无关
    rec = recall_score(y_true, preds)          # 灵敏度
    spec = tn / (tn + fp)                      # 特异度
    f1 = f1_score(y_true, preds)               # F1
```

**为什么 TTA 只用水平翻转？** 超声探头左右扫查在物理上等效——翻转后的图像仍然是"合法"的超声图像。垂直翻转不合理（探头方向固定），旋转过多可能改变病灶纵横比（乳腺超声中纵横比>1 是恶性特征）。水平翻转是最安全、最便宜的 TTA，稳定提升 AUC 约 0.003-0.005。

**为什么做多阈值评估？** AUC 是 threshold-independent 的（只衡量排序能力），但实际临床使用需要一个具体阈值。我们评估 4 个阈值（0.40 偏 Recall、0.50 均衡、0.55 偏 Specificity），让医生根据场景选择——筛查场景用低阈值（宁可误报不可漏诊），确诊场景用高阈值。

#### 融合机制详解

**模型注册表 `MODELS_REGISTRY`：**

```python
MODELS_REGISTRY = {
    'V2.3':  {'cls': BreastCancerMultiTaskNet,      'weight': 'best_model_v2.3.pth',       'arch': 'v2'},
    'V5.1':  {'cls': V5Model,                        'weight': 'best_model_v5.1.pth',       'arch': 'v5'},
    'V5.2':  {'cls': V5ModelUNetPP,                   'weight': 'best_model_v5.2.pth',       'arch': 'v5'},
    'V6':    {'cls': BreastCancerMultiTaskNetV6,      'weight': 'best_model_v6.pth',         'arch': 'v6'},
    'SAM':   {'cls': BreastCancerMultiTaskNet,        'weight': 'best_model_v7.6.pth',       'arch': 'v2'},
    'S123'/'S456'/'S789': ...                        'arch': 'v2'},
}
```

**不同架构的输出提取 `extract_cls_logits()`：**

```python
def extract_cls_logits(model, images, arch):
    out = model(images)
    if arch == 'v2':   return out[1]      # V2.3: (mask, cls) → cls
    elif arch == 'v5': return out[1]      # V5.x: (mask, cls, birads) → cls
    elif arch == 'v6': return out[1]      # V6:   (seg_list, cls) → cls
```

**融合策略：简单平均 vs Stacking：**

- **简单平均**：所有模型的 sigmoid 概率直接求均值。在 V7.3 中证明优于 Stacking。
- **Stacking**：在验证集上用 Logistic Regression 学习 `w₁·logit₁ + w₂·logit₂ + w₃·logit₃ + b`。在 368 样本的验证集上严重过拟合（给了最弱的 V5.2 最高权重），AUC 0.9052 < 0.9073。

#### 融合模型的选择逻辑

7 个可用模型中，融合需要挑选：
- **V2.3（必须）**：AUC 0.903，冠军单模型，融合基石
- **V5.1（推荐）**：BI-RADS 辅助头，错误模式与 V2.3 部分互补
- **V5.2（推荐）**：UNet++，虽然过拟合但错误模式不同
- **SAM（推荐）**：AUC 仅 0.892 但贡献了最大单次融合提升（+0.002）
- **S123/S456/S789（可选）**：3 个种子模型共贡献 +0.0002，性价比极低
- **V6（不推荐）**：AUC 0.872 拖累融合

**最佳实践：4 模型（V2.3+V5.1+V5.2+SAM）AUC 0.9093，推理时间约 3× 单模型。**

## 四、31 个实验完整迭代史

### V1：基线建立（1 实验）

**做了什么：** ResNet34 + U-Net + FocalLoss(α=0.7,γ=2) + BCE 分割 + 256×256 Resize + TTA 水平翻转 + 样本级随机划分

**结果：** Val AUC 0.959, Test AUC 0.905

**教训：** Val AUC 比 Test AUC 高 0.054——巨大的泛化差距。根因是**样本级划分导致患者数据泄露**。同一患者的左右侧图像分散在 train/val 中，模型学到了"患者特征"而非"病灶特征"。这是后续所有实验的起点。

---

### V2：修复数据泄露 + 单变量增量（5 实验）

#### V2.0 — 全改崩溃
**改动：** EfficientNet-B3 + BI-RADS 输入 + Dice 分割损失 + RandomResizedCrop 多尺度 + 患者级划分 + 多尺度 TTA

**结果：** AUC 0.857，Recall 0.629（暴跌）

**教训：** 多个改动同时上线 = 无法定位问题。BI-RADS 作为输入特征在测试集上用占位符 0，模型学到"BI-RADS=0 → benign"。"一次只改一个变量"成为后续实验的铁律。

#### V2.1 — 384×384 Resize
**改动：** 单变量：256→384 固定 Resize

**结果：** AUC 0.888（↓0.015 vs V1）

**教训：** 更高分辨率在 1507 样本上导致过拟合。ResNet34 的参数量在 384×384 下产生了更多激活，但训练样本不足以约束。

#### V2.3 — 回退 256×256 + CosAnnealing ✅
**改动：** 384→256 Resize + CosineAnnealingLR + 患者级划分

**结果：** AUC **0.9033**（此后 22 个实验的基线）

**为什么 256×256 是最优的？** 这是"压缩-泛化"权衡的甜点。256×256 对 321px 平均宽度的训练集压缩约 20%，这种适度的压缩起到了正则化作用——模型被迫学习对尺度鲁棒的特征。384×384 压缩太少，过拟合；更小尺寸（如 128×128）信息损失太大。

#### V2.4 — BI-RADS 样本级损失权重
**改动：** BI-RADS 2→weight=0.8, 3→0.7, 4→1.3, 5→2.0

**结果：** AUC 0.867（暴跌）

**教训：** 人为操纵损失权重让模型过度关注 BI-RADS 4/5 的样本。模型在这些"高权重样本"上过拟合，丧失了泛化能力。**让模型自主学习比给先验权重更有效。**

#### V2.5 — RandomResizedCrop 多尺度训练
**改动：** 固定 Resize → RandomResizedCrop(0.8-1.0, ratio=0.95-1.05)

**结果：** AUC 0.877

**教训：** 即使是窄 ratio、高 min_scale 的"温和"随机裁剪，训练管道和测试管道的差异仍然损害泛化。**训练和测试的数据处理必须完全一致。**

---

### V3：辅助任务探索（9 实验，35 epochs 统一标准）

#### V3.1-V3.3 — 信息瓶颈方法
- V3.1：Stage1 纯分割 → 冻结编码器 → Stage2 分类。AUC 0.853
- V3.2：预测掩膜 + 原图 → 4 通道输入 → 分类。AUC 0.825
- V3.3：BBOX ROI 裁剪 → 两阶段推理。AUC 0.847

**统一教训：分割特征 ≠ 分类特征。** 冻结分割编码器后再接分类头，分类头无法重新组织底层特征。ROI 裁剪丢失了病灶周围组织的上下文信息（肿瘤对周围组织的浸润是恶性关键特征）。

#### V3.4-V3.6 — 辅助预测头
- V3.4：BBOX 检测辅助头。AUC 0.887
- V3.5：Histology 病理亚型辅助头（27 类）。AUC 0.888
- **V3.6：BI-RADS 辅助预测头（4 类）。AUC 0.899, F1 0.787**（单模型 F1 最高）

**为什么 BI-RADS 辅助头有效？** 在训练时，模型不仅要回答"良性还是恶性"，还要回答"这看起来像 BI-RADS 几级"。BI-RADS 的视觉特征（边缘是否规则、有无钙化、回声模式）与良恶性判断高度相关。这个辅助任务强制编码器学习这些判别性特征。

#### V3.7-V3.9 — 优化与融合
- V3.7：EMA + SE-ResNet50。AUC 0.892（更大模型+EMA=双重过拟合）
- V3.9：5-fold CV 融合。AUC 0.893（同质模型融合收益极小）

---

### V4：组合实验（5 实验，35 epochs）

- V4.1：BI-RADS 头 + Tversky(β=2)。AUC 0.875（Tversky 过罚 FN）
- V4.2：UNet++。AUC 0.887（Val 0.940！史上最大过拟合）
- V4.3：UNet++ + BI-RADS + Tversky。AUC 0.871（组合最差）
- V4.4：DenseNet121。AUC 0.885
- V4.5：EfficientNet-B1。AUC 0.872

**统一结论：更强的架构/更复杂的损失，在 1507 样本上一律过拟合。**

---

### V5：异质模型融合（2+2 实验，35 epochs）

#### V5.1-V5.2 — 训练异质架构
- V5.1：V2.3 + BI-RADS 辅助头 + SWA。AUC 0.893（SWA 无效）
- V5.2：V2.3 + UNet++ + SWA。AUC 0.893（SWA 无效）

**SWA 为什么无效？** 1507 样本上 35 epoch 的训练轨迹没有足够的"平坦区域"来让权重平均产生增益。UNet++ 在 epoch 16 后就开始过拟合，平均 epoch 28-35 的权重反而包含了过拟合的模型。

#### V5 融合 — 突破 0.903 墙
- V2.3+V5.1（2 模型）：AUC **0.9057**
- V2.3+V5.1+V5.2（3 模型）：AUC **0.9073** 🎉

**这是 22 个实验后首次超越 V2.3 的单模型 AUC！** 三个模型的架构不同（UNet / UNet+BI-RADS / UNet++），它们的错误模式不完全重叠，简单概率平均就能产生增益。

### V6：LoMix 多尺度组合损失（1 实验）

**做了什么：** V2.3 + 4 尺度分割头 + CombinatorialMutationsLossModule（4 原始 + 11 组合 × 4 算子 = 15 个分割损失项）

**关键观察：** 训练中 LoMix 的可学习权重从 softplus(0)=0.693 被持续推低到 ~0.62。模型在"抗议"过多的分割监督。

**结果：** Val AUC 0.933（历史第三），Test AUC 0.872（历史倒数）。泛化差距 **0.061**。

**教训：** 分割损失项的数量与泛化能力呈负相关（1项 BCE=0.903, 4项简化=0.880, 15项 LoMix=0.872）。在 1507 样本上，每个额外的像素级监督信号都在消耗宝贵的模型容量。

### V7：融合优化 + 多样性扩增（6 实验）

#### V7.1-V7.3 — 零成本优化（全部失败）

#### V7.5 — Seed Ensemble
3 个不同随机种子（123/456/789）训练 V2.3：
- S123: Val 0.948, Test 0.885
- S456: Val 0.911, Test 0.898
- S789: Val 0.931, Test 0.885

**SAM 单独不优但融合贡献巨大——这是整个项目最重要的发现之一。** SAM 在 loss landscape 中找到了一个不同的平坦极小值，该处的错误模式与 AdamW 训练的模型有显著差异。**融合的收益来自错误模式的多样性，而非单个模型的绝对性能。**

## 五、核心洞察与设计哲学

### 5.1 数据规模是唯一的硬约束

1507 张训练样本决定了模型复杂度的天花板。这不是调参能改变的——是信息论意义上的。在 31 个实验中：
- 更强的 encoder（SE-ResNet50/DenseNet121/EfficientNet-B1/B3）→ **全部过拟合**
- 更复杂的架构（UNet++）→ **Val 0.940 / Test 0.887（过拟合）**
- 更复杂的损失（Dice/Tversky/LoMix）→ **一概不如 BCE**
- 更高的分辨率（384×384）→ **过拟合**
- 更复杂的训练策略（RandomResizedCrop/MixUp）→ **泛化更差**

**ResNet34 + U-Net + BCE + 256×256 就是信息论最优。** 这不是因为 ResNet34 比 EfficientNet 好——是因为 EfficientNet 在 1507 样本上过拟合了。

### 5.2 损失函数复杂度与泛化能力呈负相关

| 分割损失 | 损失项数 | Test AUC | Val-Test Gap |
|----------|----------|----------|--------------|
| BCE (V2.3) | 1 | 0.903 | 0.021 |
| 简化多尺度 BCE (V7.4) | 4 | 0.880 | 0.052 |
| LoMix 组合 (V6) | 15 | 0.872 | 0.061 |

每个额外的分割监督信号都在消耗模型的泛化容量。在小数据集上，**简单 = 好**。

### 5.3 融合的收益来自错误模式多样性，而非模型绝对性能

| 添加的模型 | 单模型 AUC | 融合后 AUC | Δ |
|-----------|-----------|-----------|-----|
| V5.1 | 0.893 | 0.9057 | +0.0024 |
| V5.2 | 0.893 | 0.9073 | +0.0016 |
| SAM | **0.892** | **0.9093** | **+0.0020** |
| S456 | 0.898 | 0.9073 | +0.0000 |

SAM 单独最弱（0.892），但贡献了第二大的融合提升（+0.002）。S456 单独最强（0.898），但对融合几乎无增益。**选择融合模型的标准不是"谁最强"，而是"谁最不同"。**

### 5.4 训练/测试管道一致性 > 一切花哨技巧

V2.5 用窄 ratio(0.95-1.05) 和高 min_scale(0.8) 的 RandomResizedCrop，试图在"几乎不改变纵横比"的前提下引入多尺度。结果：AUC 0.877（↓0.026 vs V2.3）。训练和测试阶段的任何差异都会被模型"记住"并放大。

V7.2 在推理时用 0.75×/1.0×/1.25× 多尺度 TTA，结果 AUC 反降。模型只在 256×256 上训练过，在 192×256 或 320×256 上推理结果不可靠。

**核心原则：训练管道的每个操作，测试管道必须精确复制。Resize 固定的 256×256 是唯一正确的选择。**

### 5.5 医学影像划分的金科玉律

患者级划分修复前：Val AUC 0.959 → Test 0.905（虚高 0.054）  
患者级划分修复后：Val AUC 0.924 → Test 0.903（虚高 0.021）

**永远按患者/检查/序列分组，永远不要按单张图像随机划分。** 这在所有医学影像任务中都是第一优先级。

### 5.6 V3.6 BI-RADS 辅助预测头是单模型最佳创新

在所有 31 个实验中，唯一在单模型层面接近 V2.3 的只有 V3.6（AUC 0.899 vs 0.903）。而且 V3.6 的 F1（0.787）反超了 V2.3（0.761）。

**核心机制：** 辅助预测头（让模型同时预测 BI-RADS 等级）比辅助输入（把 BI-RADS 喂给模型）和辅助损失权重（按 BI-RADS 调 loss 权重）都更有效。因为：
- 辅助输入：测试集没有 BI-RADS → 占位符 0 误导模型
- 辅助权重：人为先验 → 过拟合于特定 BI-RADS 等级的样本
- 辅助预测头：只在训练时施加额外监督 → 编码器学会 BI-RADS 相关的视觉特征 → 间接增强良恶性判断

### 5.7 优化技巧在极端小数据下的失效

| 技巧 | 理论优势 | 我们的结果 | 为什么 |
|------|---------|-----------|--------|
| SWA | 平均权重 → 平坦极小值 | 无效 | 35 epoch 不够探索平坦区域 |
| EMA | 指数平滑 → 更稳定 | 无效 | 同上 |
| SAM | 显式寻求平坦区 | 单模型无效，融合有效 | 找到了不同的极小值 |
| CosAnnealing | 平滑衰减 | ✅ 有效 | 比 StepLR 更稳定 |

### 5.8 FocalLoss 的参数选择

- `α=0.7`（不是标准 0.75）：我们刻意让恶性权重略低于标准值（0.7 vs 0.75），因为测试发现 α=0.75 时模型过度偏向恶性（Recall 虚高但 Specificity 崩塌）
- `γ=2`（标准值）：恰好区分难易样本，γ 更大（3-5）会过度抑制易样本的学习，γ 更小（0-1）退化到接近标准 BCE

### 5.9 256×256 为什么是最优分辨率？

这不是巧合。训练集平均宽度 321px，256px 的压缩比是 321/256=1.25。这个压缩比：
- 足够大以提供正则化（防止过拟合于图像细节）
- 足够小以保留病灶形态（最小病灶约 240px，压缩后约 192px，仍可分辨）
- 384×384（比 1.18）压缩太少 → 过拟合
- 128×128（比 2.5）压缩太多 → 信息损失

### 5.10 项目的最终推荐方案

**纯分类 (V9):**
```
推理: python test_ensemble.py --models DISTILL,SOUP,V2.3 --device cuda --fusion logit_mean --weights 0.04342,0.87861,0.07797
AUC:  0.9139
阈值: 0.55 (F1=0.7952) / 0.545 (best F1=0.7981)
推理: 3 模型 × TTA flip × 2 = 6 次前向传播
```

**分割→分类联合 (V10, 🏆 当前最佳):**

阈值: 0.55 (F1=0.7971)
推理: 分割 (3 模型 × TTA) + 分类 (3 模型 × TTA × 2 路[全图+ROI]) ≈ 18 次前向传播

V10 的 AUC 比 V9 高 0.86%（测试集）~1.23%（外部数据），Sensitivity 提升尤其显著（+1.9%~+4.4%）。代价是推理时间约为 V9 的 3 倍。如追求极致性能用 V10，如追求速度用 V9。

### 7.3 常见问题

**Q: 如何测试单模型？**

A: `src/evaluation/test_single.py` 是单模型测试脚本，当前配置为 V6 模型。如需测试 V2.3，修改其中的模型导入和权重路径。

**Q: 为什么有 `BreastCancerMultiTaskNet` 和 `BreastCancerMultiTaskNetV6` 两个类？**

A: V2-V5 的实验使用 `src/models/base.py` 中的简单架构（SMP 内置分类头）。V6 需要多尺度分割输出，创建了 `src/models/v6.py` 中的新架构（手动遍历 decoder.blocks）。两者不兼容——V2.3 的权重不能加载到 V6 架构上。`src/evaluation/test_ensemble.py` 通过 `arch` 字段处理这种差异。

**Q: LoMix 的 `CombinatorialMutationsLossModule` 还能用吗？**

A: 代码保留了完整的 LoMix 模块（`src/losses/lomix.py`），但在当前 1507 样本的数据规模下不建议使用。如果在更大的超声数据集上（>5000 张），可以重新激活。需配合 `src/models/v6.py` 的多尺度架构使用。

**Q: 如果我想继续提升 AUC，应该怎么做？**

A: 当前天花板约 AUC 0.91-0.92。剩余可能的方向：
1. 混合更多异质优化器训练的模型（除 SAM 外，还可尝试 RAdam、LAMB 等）
2. 获取更多训练数据（最可靠但不可控）
3. 测试时用 MC Dropout（开 Dropout 跑 5 次取平均——但我们试过 SWA/EMA 无效，可能也不乐观）
4. 外部数据预训练（如在大规模超声数据集上预训练 encoder）

### 8.6 下一阶段“跳出现有框架”的方向

- 纯分类模型：去掉 decoder，仅用分类 backbone；mask 面积、长宽比、病灶占比作为结构化特征后融合。
- ROI + 全图双分支：全图保留上下文，ROI 强化病灶边缘和内部回声。
- 轻量新 backbone：ConvNeXt-Tiny、MobileNetV3、EfficientNetV2-S，严格控制容量，不再盲目上大模型。
- 自监督预训练：用全部可见图像做 SimCLR/DINO 风格 encoder 预训练，再微调分类。
- 概率校准：对 SOUP 或 DISTILL+SOUP 做 temperature/bias calibration，目标是改善固定阈值下的 F1/Recall。

## 十一、2026-06-12 V10 分割→分类联合推理 🏆

### 11.1 为什么又有 V10

V8 的"ROI 双分支"是端到端训练的——分类器和分割器耦合在一起，彼此牵制。V10 换思路：**把分割和分类完全解耦**。用独立训练的高精度 MiT-B2 UNet 分割模型定位病灶，然后把 ROI 送进已有 V9 分类器。

类比：V8 是让一个实习生同时学会看片子+画框；V10 是先让专家（分割模型）画好框，再让确诊医生（V9）只看框里的东西。

### 11.2 管线详解

```
图像 → Resize 256 → V9 三模型 → prob_original ─┐
                                                 ├→ 0.5:0.5 融合 → 恶性概率
图像 → 分割管线 (MiT-B2 ×3) → 掩膜 → ROI → V9 三模型 → prob_roi ─┘
```

分割管线:
  1. 粗分割: 2 个 MiT-B2 UNet 各自 TTA → 平均 (384×384)
  2. ROI 提取: 粗掩膜>0.5 → 正方形扩边 (margin=0.35)
  3. ROI 精修: 1 个 MiT-B2 UNet TTA → 放回全图
  4. 融合: 0.75×粗 + 0.25×精 → 阈值 0.38 → 后处理

### 11.5 下一步建议

V10 已经证明了解耦分割→分类路线的有效性。后续可以从以下方向继续提升：

1. **更强的分割模型**：当前分割在外部数据 Dice 仅 0.48，换用更大 encoder 或域适应训练可进一步提升分类。
2. **动态融合权重**：当前 0.5:0.5 是固定值。可以根据分割置信度或 ROI 面积动态调整权重。
3. **多尺度 ROI**：不同 margin 的 ROI 可能捕获不同程度的上下文信息。
4. **分割质量感知**：如果分割 Dice 可以自估计，低质量分割时可降低 ROI 支路权重。
