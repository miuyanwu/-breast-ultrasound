## 一、赛题与数据

### 1.1 任务定义

| 子任务 | 类型 | 评估指标 |
|--------|------|----------|
| 分类（主） | 二分类 benign/malignant | AUC, Accuracy, Sensitivity, Specificity, Precision, F1 |
| 分割（辅） | 像素级病灶掩膜 | 辅助训练，不直接评分 |

### 1.2 数据规模

| 集合 | 良性 | 恶性 | 总计 | 不平衡比 |

- 病理类型：恶性 10 类（浸润性导管癌为主 520/607），良性 17 类（纤维腺瘤为主 835/1268）

## 二、最终方案架构

### 2.0 当前最佳推理管线（V10 分割→分类, AUC 0.9225）🏆

详见 §十。核心思路：独立分割模型定位病灶 → ROI 区域 V9 分类 → 与全图 V9 概率 0.5:0.5 融合。零训练成本（复用 V9 权重），测试集 AUC +0.86%，外部数据 AUC +1.23%。

### 2.1 V9 纯分类推理管线（AUC 0.9139）

```
测试图像 [H, W, 3]
    │
    ├─ Resize(256, 256)
    ├─ TTA: 原图 + 水平翻转 ──────────────────────────┐
    │                                                │
    ├─ SOUP  (Greedy Soup of 5 ResNet34 U-Nets) ─► logit₁  (w=0.8786)
    ├─ DISTILL (4教师蒸馏到 ResNet34 U-Net) ───────► logit₂  (w=0.0434)
    ├─ V2.3  (ResNet34+U-Net 基线) ───────────────► logit₃  (w=0.0780)
    │                                                │
    └─ 加权 logit 融合 ───────────────────────────────┘
         p = sigmoid(Σ wᵢ · logitᵢ)
                    │
              阈值 0.55 → benign/malignant
```

> V9 前身是 4 模型异质融合 (V2.3+V5.1+V5.2+SAM, 修正后 AUC=0.9072)。经过蒸馏+权重汤压缩为 3 个同架构模型，再通过 logit 空间加权融合达到当前最高分。

### 2.2 V9 运行所需文件

V9 三个模型（SOUP/DISTILL/V2.3）共用 `BreastCancerMultiTaskNet` 架构：

| 文件 | 作用 |
|------|------|
| `src/evaluation/test_ensemble.py` | 推理入口：加载模型 → TTA 翻转 → 加权 logit 融合 → 评测 |
| `src/config.py` | 模型注册表 + `resolve_weight_path()` + 融合权重预设 |
| `src/models/base.py` | `BreastCancerMultiTaskNet`（V2.3 架构，V9 三模型共用） |
| `src/data/dataset.py` | `BreastUltrasoundDataset` 测试集加载 |
| `weights/best_model_soup.pth` | ~98MB，权重建模 0.8786（主排序器） |
| `weights/best_model_distill_seed42.pth` | ~98MB，权重 0.0434（边界修正） |
| `weights/best_model_v2.3.pth` | ~98MB，权重 0.0780（边界修正） |

> 注：`src/config.py` 模块顶层导入了 `src/models/v5.py` 和 `src/models/v6.py`，所以这两个文件当前也必须存在才能 import，虽然 V9 推理不碰它们。可改为懒加载优化。

### 2.2 单模型架构（V2.3 基线）

```
输入 [B, 3, 256, 256]
    │
    ▼
┌──────────────┐
│  Encoder     │  ← ResNet34 (ImageNet pretrained)
│  (共享)      │
└──┬───────────┘
   │
   ├──► Decoder ──► SegHead ──► mask_logits [B,1,256,256]
   │                        ↕ BCEWithLogitsLoss (×1.5)
   │
   └──► GlobalAvgPool ──► Dropout(0.4) ──► cls_logits [B]
                                            ↕ FocalLoss(α=0.7,γ=2)
```

### 2.3 训练配置

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 编码器 | ResNet34 (ImageNet pretrained) | 1507样本下的最优容量 |
| 架构 | SMP U-Net + 分类头 | 多任务学习 |
| 优化器 | AdamW (lr=1e-4, wd=1e-4) | |
| 调度器 | CosineAnnealingLR (T_max=35, η_min=1e-6) | |

## 三、实验迭代史（34 个实验）

### V1：基线建立（1 实验）
- ResNet34 + U-Net + FocalLoss + 256×256 + 样本级划分
- AUC 0.9054，但 Val AUC 0.959 → 0.905 的巨大差距揭示**患者级数据泄露**
- 811/1064 Case 有左右侧配对图像

### V2：修复数据泄露 + 增量优化（5 实验）
- V2.0：全改 → AUC 0.857（崩溃，多项改动互相干扰）
- V2.1：384×384 → AUC 0.888（分辨率过高，过拟合）
- **V2.3：回退 256×256 → AUC 0.9033** ✅（此后所有实验的基线）
- V2.4：BI-RADS 损失权重 → AUC 0.867（严重过拟合）
- V2.5：RandomResizedCrop → AUC 0.877（管道不一致）

### V3：辅助任务探索（9 实验）
- V3.1–V3.3：两阶段/Mask-Guided/BBOX ROI → 均 < 0.855
- V3.4–V3.5：BBOX/Histology 辅助头 → 0.887–0.888
- **V3.6：BI-RADS 辅助预测头 → AUC 0.899, F1 0.787**（单模型 F1 最高）
- V3.7：EMA + SE-ResNet50 → AUC 0.892（更大模型过拟合）
- V3.9：5-fold 同质融合 → AUC 0.893

### V4：组合实验（5 实验）
- V4.2：UNet++ → AUC 0.887（Val 0.940，严重过拟合）
- V4.1/3/4/5：Tversky/DenseNet/EfficientNet → 0.871–0.885
- **结论：更强的架构/损失无一超越简单基线**

### V5：异质融合突破（2+2 实验）
- V5.1：V2.3 + BI-RADS 头 + SWA → AUC 0.893（SWA 无效）
- V5.2：V2.3 + UNet++ + SWA → AUC 0.893（SWA 无效）
- **V2.3+V5.1 两模型融合 → AUC 0.9057**（首破 0.903）
- **V2.3+V5.1+V5.2 三模型融合 → AUC 0.9073** 🥇（当时纪录）

### V6：LoMix 多尺度损失（1 实验）
- 4 尺度 + 4 种融合算子 = 15 个分割损失项
- Val AUC 0.9330，Test AUC **0.8723**（gap 0.061）
- LoMix 可学习权重被模型主动压低（0.69→0.62），但过量监督仍然过拟合
- **结论：复杂损失函数在小数据集上一致失败**

### V7：融合优化 + Seed + SAM（6 实验）
- V7.1 四模型(+V6)：AUC 0.9063 ❌（弱模型拖累）
- V7.2 多尺度 TTA：AUC 0.9064 ❌（非训练尺度推理有害）
- V7.3 Stacking：AUC 0.9052 ❌（小验证集过拟合）
- V7.4 简化多尺度：AUC 0.8802 ❌（多尺度架构本身过拟合）
- **V7.5 Seed Ensemble：6 模型 AUC 0.9088** ✅
- **V7.6 SAM：4 模型 AUC 0.9093** ✅✅
- **7 模型终极融合：AUC 0.9095** 🏆（旧公式记录；修正后 7 模型不再优于 4 模型）

### V8：跳出 U-Net 框架（3 实验） ❌ 全部失败
- V8.1 纯分类全图：去掉 decoder，仅用分类 backbone → AUC 0.8692（远低于 compact）
- V8.2 预测 ROI + 全图双分支：全图保留上下文 + ROI 强化病灶边缘 → AUC 0.8756
- V8.3 ROI 双分支 + shape 特征：Val 0.9018 但 Test 仅 0.8752（严重过拟合）
- **结论：换纯分类/ROI 结构没有提供新泛化信号，不建议继续**

### V9：稀疏加权 logit 融合（1 实验） 🏆 当前最佳
- 把最强 compact 信息压缩到 3 个同架构模型 (SOUP/DISTILL/V2.3)，在 logit 空间加权
- 融合公式：p = sigmoid(0.0434·logit_DISTILL + 0.8786·logit_SOUP + 0.0780·logit_V2.3)
- SOUP 是主排序器（权重 87.9%），DISTILL 和 V2.3 只管边界修正
- **AUC 0.9139，best F1 0.7981@0.545** ✅✅

## 四、关键洞察

### 4.1 数据规模决定模型复杂度上限

1507 张训练样本是核心约束：
- **ResNet34 是最优 encoder**（更强的全部过拟合）
- **UNet 是最优架构**（UNet++ 的额外参数导致过拟合）
- **256×256 是最优分辨率**（384×384 过拟合）
- **BCE 是最优分割损失**（Dice/Tversky/LoMix 均不如）

### 4.2 融合收益来自错误模式多样性

| 融合方式 | 模型数 | AUC | 关键 |
|----------|--------|-----|------|
| 5-fold 同质 | 5 | 0.893 | 模型太相似, 负收益 |
| 3 模型异质 | 3 | 0.907 | 架构多样性 |
| 4 模型(+SAM) | 4 | 0.907 | SAM不同极小值(修正后) |
| V9 稀疏加权 | 3 | **0.914** | logit 空间加权, 当前最佳 |

> 历史 7 模型 0.9095 为旧公式记录；修正后不再优于 4 模型。**SAM 单独 AUC 仅 0.892，但加入融合后贡献了最大单次提升（0.907→0.9072 修正后）。** 弱模型 ≠ 无用模型——关键是其错误模式是否与现有模型不同。

### 4.3 损失函数复杂度与泛化呈负相关

| 分割损失 | 损失项数 | Test AUC | Gap |
|----------|----------|----------|-----|
| BCE (V2.3) | 1 | 0.903 | 0.021 |
| 简化多尺度 BCE (V7.4) | 4 | 0.880 | 0.052 |
| LoMix 组合 (V6) | 15 | 0.872 | 0.061 |

每个额外的分割监督信号都在 1507 样本上增加过拟合风险。

### 4.4 数据泄露的隐蔽性

患者级 vs 样本级划分导致 Val AUC 差 0.035。811/1064 Case 有左右侧配对。**医学影像中必须先按患者/检查分组再划分。**

### 4.5 优化技巧的局限性

| 技巧 | 效果 |
|------|------|
| SWA | 两个实验均无效 |
| EMA | 无效 |
| SAM | 单模型不优，但融合贡献大 |
| CosineAnnealing | ✅ 稳定收敛 |

## 七、2026-06-09 Compact 两模型新最佳

### 7.1 为什么要更新结论

复查融合代码时发现历史 `src/evaluation/test_ensemble.py` 的”概率平均”存在运算优先级问题，旧实现实际计算的是 `1 / mean(1 + exp(-logit))`，不是严格的 `mean(sigmoid(logit))`。修正为 `--fusion prob_mean` 后重新评估，历史 4/7 模型分数需要更新。

### 7.2 修正后结果

| 方案 | 模型数 | AUC | 代表阈值表现 |
|------|------:|----:|-------------|
| V2.3+V5.1+V5.2+SAM | 4 | 0.9072 | F1=0.7748 @0.55 |
| V2.3+V5.1+V5.2+S123+S456+S789+SAM | 7 | 0.9063 | F1=0.7832 @0.55 |
| SOUP 权重汤单模型 | 1 | 0.9083 | F1=0.7866 @0.55 |
| DISTILL 蒸馏单模型 | 1 | 0.9074 | F1=0.7773 @0.55 |
| **DISTILL+SOUP compact 融合** | **2** | **0.9114** | **F1=0.7832 @0.55** |

结论：不需要 7 模型。两模型 `DISTILL+SOUP` compact 融合 AUC=0.9114 已超修正后 4/7 模型；如果必须单模型，优先使用 `SOUP` 或 `DISTILL`。此节撰写后被 §九 V9 三模型加权 logit (AUC=0.9139) 超越，当前最佳见 §九。

### 7.3 新增代码与权重

- `src/training/train_distill.py`: 用 V2.3/V5.1/V5.2/SAM 教师融合输出训练一个 ResNet34 U-Net student。
- `scripts/make_model_soup.py`: 对同架构 checkpoint 做 greedy model soup，当前 soup 成员为 V2.3、SAM、S123。
- `scripts/search_fusion_weights.py`: 小规模融合搜索工具，默认只搜索 DISTILL/SOUP/V2.3，避免大搜索长时间卡住。
- `src/evaluation/test_ensemble.py`: 支持 `--fusion prob_mean/logit_mean/legacy_bug`、`--device auto/cuda/cpu`、`--weights`、`DISTILL`、`SOUP`。
- `src/inference/engine.py` / `app/main.py`: `compact` 模式优先加载 `DISTILL+SOUP+V2.3` weighted logit，缺权重时自动回退。

本地新增权重：

### 7.5 后续跳出现有框架的候选方向

- 纯分类轻量模型：去掉 U-Net decoder，只用 mask 面积/形状统计作为辅助特征，验证是否能减少多任务牵制。
- ROI + 全图双分支：一支看全图上下文，一支看 mask/BBOX 附近病灶区域，再做 late fusion。
- 新 backbone：优先尝试 ConvNeXt-Tiny、MobileNetV3、EfficientNetV2-S，小模型强正则，不再盲目加大容量。
- 自监督预训练：用训练图像和测试图像做无标签 encoder 预训练，再微调分类。
- 概率校准：对 SOUP 做 temperature/bias calibration，目标是在保持 AUC 的同时改善 0.45/0.50 默认阈值表现。

### 7.6 V8 跳出 U-Net 框架实验结果

已实现 `experiments/v8/`，验证纯分类、预测 ROI + 全图双分支、ROI+shape 特征三条路线。结果均未超过 compact / V9 weighted logit。

| 版本 | 方案 | Val AUC | Test AUC | 结论 |
|------|------|--------:|---------:|------|
| V8.1 | 纯分类全图模型 | 0.8929 | 0.8692 | 明显低于 compact |
| V8.2 | 预测 ROI + 全图双分支 | 0.8956 | 0.8756 | ROI 带来小幅提升但仍不足 |

## 九、2026-06-09 V9 稀疏加权 logit 融合

V8 说明“换纯分类/ROI 双分支结构”没有提供新泛化信号，因此下一轮转向校准与稀疏融合。使用已有最强 compact 相关模型 `DISTILL,SOUP,V2.3` 做小范围权重搜索，找到新的当前最高分。

| 方案 | 模型数 | 融合 | Test AUC | 代表阈值表现 |
|------|------:|------|---------:|-------------|
| DISTILL+SOUP compact | 2 | prob_mean | 0.9114 | F1=0.7832 @0.55 |
| **DISTILL+SOUP+V2.3 weighted compact** | **3** | **weighted logit** | **0.9139** | **F1=0.7952 @0.55; best F1=0.7981 @0.545** |

最佳权重：

## 十、2026-06-12 V10 分割→分类联合推理 🏆 当前最新最佳

### 10.1 动机

V8 在"ROI 双分支"方向上失败，但 V8 做的是**端到端训练**的 ROI 分支——模型需要同时学会分割和分类。V10 换了一个思路：**把分割和分类解耦**。用独立训练的高精度分割模型（MiT-B2 UNet）先定位病灶，再把 ROI 区域送入已有的 V9 分类器。

核心差异：
```
V8:  图像 → [端到端训练] → 同时输出分割 + ROI分类 + 全图分类 → 融合
V10: 图像 → [预训练分割模型] → ROI → [已有V9分类器] → 与全图概率融合
```

### 10.2 管线架构

```
测试图像 [H, W, 3]
    │
    ├─ Resize(256,256) → V9 三模型加权 logit → prob_original (50%)
    │
    ├─ 两阶段分割管线 (MiT-B2 UNet ×3):
    │   ├─ 粗分割: 2× MiT-B2 UNet 平均 → Dice 0.76 (测试集)
    │   ├─ ROI 提取: 粗掩膜 → 正方形扩边裁剪 (margin=0.35)
    │   └─ ROI 精修: 1× MiT-B2 UNet → 融合 0.75×粗 + 0.25×精
    │       ↓
    │   病灶掩膜 → ROI 裁剪 (margin=1.0) → Resize(256,256)
    │       ↓
    │   V9 三模型加权 logit → prob_roi (50%)
    │
    └─ 融合: 0.5 × prob_original + 0.5 × prob_roi
              ↓
         阈值 0.55 → benign/malignant
```

### 10.5 关键发现

1. **域外泛化收益 > 域内收益**：外部数据 AUC 提升 (+1.23%) > 测试集 (+0.86%)。ROI 聚焦减轻了领域偏移。
2. **最大收益在敏感度**：外部数据 Sensitivity +4.44%，漏诊 FN 从 18→14。
3. **粗糙分割也有用**：外部数据分割 Dice 仅 0.48，但分类依然大幅提升。ROI 定位不需要完美。
4. **零训练成本**：V10 完全复用 V9 分类权重，分割模型独立训练，推理管线整合即可。
5. **与 V8 的本质区别**：V8 端到端训练让分类器受分割质量牵制；V10 解耦后各司其职，泛化更好。
